Вариант 15.
Предметная область: автоматизация маркетинга
Типы агентов: сегментация аудитории, создание рассылок, анализ отклика, оптимизация кампаний

Задание 1. Разработка полной системы из 4 агентов на Go
Реализовать  всех  агентов,  определённых  для  своей  предметной  области.  Каждый 
агент  -  отдельный  микросервис  на  Go,  общающийся  через  NATS.  Агенты  могут  быть 
разных типов (например, анализатор, валидатор, исполнитель). 

Задание 2. Цепочки задач (pipeline)
Реализовать последовательную обработку задачи несколькими агентами. Например: 
Агент  A  принимает  задачу  →  отправляет  результат  в  очередь  для  Агента  B  →  Агент  B 
обрабатывает → и т.д. Оркестратор управляет цепочкой. 

Задание 3. Распределённая трассировка (Jaeger)
Интегрировать OpenTelemetry в агенты и оркестратор. Настроить сбор трассировок 
в Jaeger (запущенном в Docker). Визуализировать полный путь выполнения распределённой 
задачи.

Задание 4. Агент с состоянием (Redis)
Реализовать агента, который сохраняет своё состояние в Redis (например, счётчики, 
статистику или кэш результатов). При перезапуске агент восстанавливает состояние.

Задание 5. Динамическое масштабирование
Реализовать систему, которая автоматически запускает дополнительные экземпляры 
агента при высокой нагрузке (например, при длине очереди > N). Использовать Docker API 
или оркестратор (Kubernetes).

Задание 6. Аукционное распределение задач
Реализовать  модель,  где  агенты  «торгуются»  за  задачу:  каждый  агент  оценивает 
свою способность выполнить задачу (cost, skill, availability). Оркестратор выбирает агента 
с наименьшей стоимостью или наилучшим соответствием.

Задание 7. Интеграция LLM-агента
Добавить в систему агента на Python, использующего LLM (локальную через Ollama 
или  облачную  через  API).  Агент  должен  выполнять  интеллектуальную  задачу  в  рамках 
предметной области (например, генерация ответа, классификация текста).

Задание 8. Веб-интерфейс для мониторинга агентов
Разработать  веб-панель  (Streamlit  или  FastAPI  +  Jinja2)  для  отображения  статуса 
агентов, очередей, результатов задач, возможности ручного запуска задач.
